Third-party product capabilities depend on `boson-sdk`. Host applications
compose them through `boson-runtime`.

## Public surface

Capability authors should import from `boson_sdk::prelude`:

- `Capability`, `CapabilityDescriptor`, `MigrationSet`, `JobHandler`
- `EventEnvelope`, `EventConsumer`, `publish_in_tx`
- `AuthenticatedUser`, `IdentityAuth`, `user_auth_middleware`
- `AdminPrincipal`, `require_scope`
- `PlatformConfig::capability_config`, `RequestContext`, `api_error`

Do not depend on adapter crates or vendor SDKs from a capability.

## Capability lifecycle

1. Declare `descriptor()`, `scopes()`, and optional `migrations()`.
2. Return pre-bound `app_router()` / `admin_router()` routers.
3. Emit events with `publish_in_tx` inside the same SQL transaction as domain
   writes.
4. Register `event_consumers()`, `job_handlers()`, and `health_checks()` as
   needed.
5. Register the capability from server, worker, and migrate binaries through
   `boson_runtime::Builder::extend`.

```rust
boson_runtime::Builder::from_env()
    .extend(|ctx, registry| {
        registry.register(Arc::new(TodosCapability::new(
            ctx.database.clone(),
            ctx.identity_auth.clone(),
            &ctx.config,
        )?))?;
        Ok(())
    })
    .run_server()
    .await
```

## Configuration

Core platform sections remain closed. Capability-owned settings live under:

```yaml
capabilities:
  todos:
    max_todos: 100
```

Deserialize with `config.capability_config::<T>("todos")`.

## Migrations

Platform migrations are embedded in `boson-runtime`. Capability migrations are
directories declared by `Capability::migrations()` and applied in registration
order. Migration version numbers must be unique across the whole database.

## Scopes

Capabilities declare Admin scopes via `Capability::scopes()`. The runtime
collects them after registration and uses the set when issuing Admin API keys.
Bootstrap tokens continue to carry `*`.

## CLI workflow

```bash
boson init my-app --boson-path .
cd my-app
boson migrate
boson dev
boson doctor
```
