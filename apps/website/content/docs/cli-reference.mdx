The `boson` CLI is the public local control plane. Run project commands from
the project root or any nested directory; Boson discovers
`.boson/project.toml` by walking upward.

## `boson init`

Create a standalone workspace:

```bash
boson init my-app
boson init my-app --path ./products/my-app
```

By default, generated `boson-runtime` and `boson-sdk` dependencies point at the
Boson Git repository and `main`. `--force` allows generated files to be written
into a non-empty destination. Internal source override flags are hidden and
intended for Boson contributors.

## `boson start`

```bash
boson start
```

The command remains attached and performs the complete local startup sequence:

1. Validate Rust, Docker, the project manifest, and configuration.
2. Start managed PostgreSQL and Dashboard services.
3. Wait for infrastructure health.
4. Build migrate, Server, and Worker packages.
5. Apply pending platform and capability migrations.
6. Start Server and Worker processes.
7. Wait for `/readyz` and the Dashboard.
8. Stream prefixed logs and supervise process health.

Ctrl+C gracefully stops the stack and preserves persistent data.

## `boson stop`

```bash
boson stop
```

Signals the active foreground supervisor and waits up to ten seconds for
graceful shutdown. If the state file is stale, Boson reconciles processes,
stops managed infrastructure, and clears the stale state.

## `boson status`

```bash
boson status
```

Reports each service's state, port, health, and Boson version. The list covers
PostgreSQL, local storage, local mail, Server, Worker, and Dashboard.

## `boson logs`

```bash
boson logs
boson logs server
boson logs worker --follow
boson logs postgres --lines 250
```

Without a service, the command prints recent PostgreSQL, Server, Worker, and
Dashboard logs. `--follow` streams new lines and `--lines` controls the initial
tail (default: 100).

## `boson doctor`

```bash
boson doctor
```

Checks project discovery, configuration, Rust, Docker, `compose.yaml`, and
local ports. Failures include a concrete `fix:` instruction. Occupied ports are
warnings because an existing Boson stack may already own them.

## `boson config`

```bash
boson config
```

Prints the effective typed configuration as formatted JSON with secrets
redacted.

## `boson update`

```bash
boson update --check
boson update
```

Checks the latest GitHub release. Without `--check`, Cargo installs the matching
`boson-cli` release tag through Cargo (`cargo install --git … --tag <tag>
boson-cli --force`).

## Removed commands

These older commands are no longer part of the public CLI:

| Removed | Replacement |
| --- | --- |
| `boson migrate` | Applied automatically by `boson start` |
| `boson dev` | `boson start` |
| `boson admin create` | Admin API / Dashboard |
| `boson overview` | Admin API `/admin/v1/overview` |
| `--server` / `--admin-token` | Removed; the CLI is no longer an Admin API client |

`boson config` now prints the local project's redacted configuration from
`.boson/config.yaml`. It does not call the Server Admin API.
