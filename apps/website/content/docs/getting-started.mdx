Create a standalone Boson application with the CLI, then iterate on a product
capability using the public Capability SDK. Boson owns the local lifecycle, so
you do not need to coordinate Docker, Cargo, migrations, and service processes
yourself.

## Prerequisites

- Rust 1.94 or newer
- Docker Desktop with the Docker daemon running
- Git

Install the CLI from the Boson repository:

```bash
cargo install --git https://github.com/Systograms/boson boson-cli
```

## Scaffold an app

```bash
boson init my-app
cd my-app
```

`boson init` writes a workspace with:

- thin `server`, `worker`, and `migrate` binaries
- a starter `items` capability
- project identity in `.boson/project.toml`
- typed local configuration in `.boson/config.yaml`
- managed local infrastructure definitions

## Start everything

```bash
boson start
```

Boson checks prerequisites, starts PostgreSQL and the Dashboard, applies
pending migrations, starts the Server and Worker, waits for health, and streams
all logs with service prefixes. Press Ctrl+C to gracefully stop everything;
persistent local data is retained.

The implementation details stay hidden: developers do not need to run Docker,
Cargo, npm, migration commands, or individual services.

## Operate

```bash
boson status
boson logs
boson logs server --follow
boson doctor
boson config
boson stop
```

`boson doctor` prints specific fixes for missing prerequisites, invalid config,
occupied ports, and unhealthy infrastructure instead of raw stack traces.

`boson config` prints the effective configuration as JSON with secrets
redacted. `boson status` reports service state, ports, health, and versions.

## Update the CLI

```bash
boson update --check
boson update
```

The updater checks the latest GitHub release and installs the matching
`boson-cli` tag through Cargo.

## Example application

The repository includes a complete standalone example at `examples/my-api`
that demonstrates authenticated routes, Admin scopes, transactional events, and
capability migrations.
