# my-api

Standalone Boson application generated by `boson init`.

## Quick start

```bash
boson start
```

Configure the infrastructure you own in `.boson/config.yaml`, then run
`boson start`. Boson validates configuration, applies pending migrations,
launches the Server and Worker, and streams unified logs. Press Ctrl+C to stop.

Useful commands: `boson status`, `boson logs`, `boson doctor`, `boson stop`.

The starter `items` capability exposes:

- `POST /v1/items` / `GET /v1/items` (authenticated end-user)
- `GET /admin/v1/items` (`items:read` Admin scope)

Edit `capabilities/items` and register additional capabilities from
`apps/server`, `apps/worker`, and `apps/migrate` through `boson_runtime::Builder::extend`.
