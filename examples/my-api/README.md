# my-api

Standalone Boson application generated by `boson create`.

## Quick start

1. Point `.boson/config.yaml` at infrastructure you own (`database.url`, and later
   storage/mail providers).
2. Verify connectivity:

```bash
boson doctor
```

3. Start Boson processes:

```bash
boson start
```

Boson validates configuration, applies pending migrations, launches the Server
and Worker, and streams unified logs. It never creates PostgreSQL, buckets, or
mail providers. Press Ctrl+C to stop.

Useful commands: `boson status`, `boson logs`, `boson doctor`, `boson deploy`,
`boson stop`.

The starter `items` capability exposes:

- `POST /v1/items` / `GET /v1/items` (authenticated end-user)
- `GET /admin/v1/items` (`items:read` Admin scope)

Register additional capabilities in `apps/app` so migrate, Server, and Worker
stay identical.
